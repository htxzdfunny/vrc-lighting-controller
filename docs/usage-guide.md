## 启动与连接

### 桌面端启动

```bash
pnpm tauri dev      # 开发模式
pnpm tauri build    # 构建发布版
```

启动后程序会：
1. 加载上次保存的灯光状态（如果有）
2. 启动 1280x720 帧合成器（60fps 循环渲染）
3. 在端口 **9000** 启动内嵌 Web 服务器
4. 启动 NDI 输出线程（需要 NDI Runtime）
5. 每 10 秒自动保存当前状态到本地 JSON

### 远端访问

同一局域网内任意设备打开浏览器，访问：

```
http://<运行软件的电脑IP>:9000
```

即可打开与桌面端完全一致的控制界面。多人可同时操作。

## 演出模式（默认主界面）

软件启动后默认显示「演出模式」。整个界面针对 **10 盏灯的现场快速操作** 设计，可全键盘操作，也支持触屏。

### 顶部条

| 元素 | 说明 |
|------|------|
| 编辑模式 | 切换是否进入"编辑/分配"状态。编辑模式开启时，按下颜色/快照槽 = 保存当前值；非编辑模式 = 应用 |
| 输出窗 | 开关独立的 90×720 抓取窗口（与备用 UI同一逻辑） |
| 备用UI | 切换到旧的多面板控制台（可在两个布局间自由切换，刷新后保留偏好） |
| 设置 | 打开设置弹窗（灯具数量 / 帧率 / NDI / Spout / 端口 等） |

### 灯光角度面板（左）

- **10 个推子**：每根推子对应 1~10 号灯。最下=黑（R=0）、最上=红（R=255），直接控制输出图像 Block A 的 R 通道（VRChat 端解码为 Tilt 角度）。
- **10 个旋钮**：指针默认指向下方，逆时针旋转增加；直接控制 Block A 的 G 通道（VRChat 端解码为 Pan 水平扫描）。
- 每根推子下方有 **sync 按钮**：开启时该灯跟随最左侧"总控"的推子/旋钮联动；关闭则独立。
- 最左侧 **总控柱**（红色）：拉动推子或旋转旋钮 = 同时写入所有 sync 开启的灯。

操作方式：

- 桌面端：鼠标拖动；推子滚轮微调；旋钮滚轮微调（按住 Shift 是更精细的步进）；旋钮双击重置到中点。
- 触屏：按住手柄上下拖；旋钮按住后绕中心画弧。

### 颜色预设面板（右）

| 控件 | 说明 |
|------|------|
| R / G / B 三推子 | 调整工作色。**编辑模式**下：仅暂存为"工作色"；**非编辑模式**下：实时写入当前选中灯 |
| M 推子 | 工作色亮度倍乘（保存到预设 / 写入灯时都会乘上） |
| 输出颜色块 | 工作色与 M 相乘后的最终预览色 |
| 10 格预设网格 | 标签 Q W E R T / A S D F G。**非编辑模式**：点击 = 把该格颜色应用到所选灯；**编辑模式**：点击 = 把当前工作色保存到该格，每格右上角的 × 按钮 = 清空回黑 |

### 灯具开关行（底部）

- 10 个数字按钮 **1 ~ 0**，颜色实时反映该灯的实际输出色（受 dimmer / strobe / 开关影响）。
- 普通模式：点一下 = 切换该灯亮/灭。
- **闪烁（按钮模式）**：开启后，按住数字按钮 = 该灯亮、松开 = 灭，可手动闪灯。
- **全选 / 多选**：在"编辑模式"下用于决定颜色推子和预设作用于哪些灯。
- 关灯只清零 Block B 输出（变成黑色），不擦除颜色记忆，重新开灯还是原色。

### 状态快照（底部右）

ZXCVBNM 共 **7 个槽位**，保存所有 10 盏灯当前的颜色 / 角度 / 开关 / 频闪。

- 普通模式：点击格子 = 切换到该状态。
- 编辑模式：点击格子 = 把当前所有灯保存到该槽；右上角 × = 清空。
- 恢复时**保留每盏灯的 sync 设置**（不会被快照覆盖），方便快速切换场景而不影响联动设置。

### 键盘映射

| 按键 | 普通模式 | 编辑模式 |
|------|----------|----------|
| `1` ~ `0` | 切换该灯开/关；按钮模式下"按住=亮，松开=灭" | 选中该灯（多选模式下追加/取消） |
| `Q W E R T A S D F G` | 应用对应颜色预设到所选灯 | 把当前工作色（含 M）保存到对应槽 |
| `Shift + Q~G` | 清空对应预设槽 | 同左 |
| `Z X C V B N M` | 切换到对应状态快照 | 切换（与普通模式同） |
| `Shift + Z~M` | 清空对应快照槽 | 把当前所有灯保存为该快照 |

文本输入框聚焦时所有快捷键自动失效。窗口失焦会自动释放按钮模式下被按住的灯键，避免"卡住亮"。

### 触屏与响应式

- 所有控件统一用 `PointerEvent`，多设备一致；旋钮支持真"转动"手势。
- 屏幕宽度 < 900px 时左右两个面板自动堆叠为上下；灯光面板内 10 通道横向滚动，总控柱钉在最左不消失。
- 屏幕宽度 < 600px 时进一步压缩，颜色预设区域纵向铺开，确保数字键和预览条始终可见。

## 左侧面板：灯具选择与编组

左侧面板纵向分为两部分，中间有细线分隔。

### Fixtures（灯具选择网格）

网格排列的按钮，每个代表一个灯具。

| 操作 | 说明 |
|------|------|
| **单击灯具按钮** | 选中该灯具（取消其他选中），按钮边框变为红色 |
| **Ctrl+单击** | 多选/取消选中某个灯具（保持已选的不变） |
| **ALL 按钮** | 选中所有灯具 |

每个灯具按钮上显示：
- **编号**：灯具序号（1, 2, 3...）
- **色点**：右侧小圆点显示该灯当前颜色

选中的灯具会成为所有控制操作的目标——改颜色、改角度、改亮度都作用于选中灯具。

### Groups（编组面板）

位于灯具选择下方。

| 元素/操作 | 说明 |
|-----------|------|
| **+ 按钮** | 展开创建编组的对话框 |
| **Group name 输入框** | 输入新编组的名称 |
| **Create 按钮** | 以当前名称和当前选中的灯具创建一个编组 |
| **编组条目** | 点击 = 选中该组内所有灯具 |
| **编组数字** | 显示该组包含的灯具数量 |
| **× 按钮** | 删除该编组 |

编组的用途：快速批量选中一组灯具。比如创建一个"前排灯"组包含灯 1-4，以后点一下就能同时控制这 4 盏灯。

## Tab 页：控制 (Control)

默认激活的页面，包含灯光的核心参数控制。

### XY Pad（Pan/Tilt 触摸板）

左侧的正方形触摸区域。

| 属性 | 说明 |
|------|------|
| **X 轴** | Pan（水平扫描），左端 = -180°，右端 = +180° |
| **Y 轴** | Tilt（垂直扫描），上端 = +180°，下端 = -180° |
| **中心交叉线** | 0° / 0° 参考线 |
| **十字光标** | 当前选中灯具的 Pan/Tilt 位置 |
| **角落度数** | 四角标注最大/最小度数 |

**操作方式：**
- 在 Pad 区域内拖动手指/鼠标 → 实时修改选中灯具的 Pan 和 Tilt
- 支持触摸屏直接拖动，拖动时会阻止页面滚动

### Color Wheel（色轮）

中间的圆形颜色选择器。

| 部分 | 说明 |
|------|------|
| **外圈色环** | 色相（Hue）0°~360°，拖动选择色相 |
| **内部方块** | 饱和度（水平，左低右高）和明度（垂直，上亮下暗） |
| **色相指示器** | 外圈上的白色小圆圈，标记当前色相位置 |
| **SV 指示器** | 内部方块上的白色小圆圈，标记当前饱和度和明度 |
| **右下预览点** | 圆形小色块，显示最终选定的颜色 |

**操作方式：**
- 拖动外圈 → 改变色相
- 拖动内部方块 → 改变饱和度和明度
- 颜色变化实时应用到选中的所有灯具

### Dimmer 推子（亮度）

右侧的竖向滑条。

| 属性 | 说明 |
|------|------|
| **范围** | 0（最暗/关灯）到 1（全亮） |
| **数值显示** | 顶部显示当前数值 |
| **标签** | 底部显示 "Dimmer" |
| **填充色** | 红色填充从底部到当前值 |

**操作方式：**
- 上下拖动手柄（水平白色横条）调整亮度
- Dimmer 乘到 RGB 颜色上输出——Dimmer = 0.5 且颜色纯白时，输出为 (128, 128, 128)

### 旋钮行（Encoder Knobs）

控制区下方的一行旋钮。

| 旋钮 | 功能 | 范围 |
|------|------|------|
| **Pan** | 水平扫描角度精确调节 | -180° ~ +180° |
| **Tilt** | 垂直扫描角度精确调节 | -180° ~ +180° |
| **Dim %** | 亮度百分比 | 0% ~ 100% |
| **Strobe** | 频闪速度（Hz） | 0 ~ 30 |

**操作方式：**
- 在旋钮上**上下拖动**改变数值（向上 = 增大，向下 = 减小）
- 桌面端可用**鼠标滚轮**微调
- 旋钮上的指示线表示当前值在范围内的位置

### Strobe ON/OFF 按钮

位于 Strobe 旋钮下方。

| 状态 | 外观 | 说明 |
|------|------|------|
| **OFF** | 灰色按钮 | 频闪关闭，灯光持续亮 |
| **ON** | 红色按钮 | 频闪开启，灯光按 Strobe 旋钮设定的速度闪烁 |

频闪逻辑在软件端实现：开启后输出画面中该灯的 Block B 会在正常颜色和黑色之间按频率切换。

## Tab 页：效果 (Effects)

用于添加自动化灯光效果。

### 效果网格

5 个效果按钮，点击激活/取消：

| 效果 | 说明 |
|------|------|
| **Color Cycle** | 颜色循环：色相按时间连续旋转，产生彩虹流动效果 |
| **Scan** | 扫描：亮度在全亮与全灭间正弦波动，像探照灯扫描 |
| **Pulse** | 脉冲：亮度按绝对值正弦波动，节奏感强 |
| **Wave** | 波浪：亮度柔和起伏，多灯间有相位差时产生波浪效果 |
| **Random** | 随机：亮度随机跳变，营造闪烁/派对氛围 |

- 按钮高亮 = 当前生效的效果
- 再次点击同一效果 = 关闭效果
- 同时只能有一个效果生效

### 效果参数

| 参数 | 说明 | 范围 |
|------|------|------|
| **Speed** | 效果速度。越大变化越快 | 0.1 ~ 10.0 |
| **Intensity** | 效果强度。0 = 无影响，1 = 完全受效果控制 | 0% ~ 100% |
| **Phase** | 相位偏移。多灯间的相位差，产生追逐/波浪效果 | 0 ~ 6.28 (2π) |

**Phase 说明：** 当 Phase = 0 时，所有灯同步变化。Phase 增大时，灯具间产生递进延迟——灯 1 先变，灯 2 稍后变，以此类推，形成追逐效果。

## 远程控制 (WebUI)

### 连接方式

1. 确保控制端电脑和远程设备在同一局域网
2. 在远程设备的浏览器中输入 `http://<控制端IP>:9000`
3. 页面加载后自动通过 WebSocket 连接
4. 连接成功后顶部状态栏显示 "Connected"

### 多人协作

- 多个设备可同时连接和操作
- 冲突策略：**后写覆盖**（Last-write-wins），最后一个修改的人生效
- 所有客户端实时同步——一端修改，所有端立即看到变化
- 顶部状态栏显示当前连接的客户端数量

### 与桌面端的区别

WebUI 与桌面端使用**完全相同的 UI 代码**。唯一的区别：
- 桌面端通过 Tauri IPC 直接调用 Rust 后端
- WebUI 通过 WebSocket 发送命令
- 输出预览在 WebUI 端不可用（帧数据通过 IPC 传输，WebSocket 不传输帧）

## 输出协议与 VRChat 接入

### 输出方式

| 方式 | 说明 |
|------|------|
| **窗口捕获** | OBS 使用"窗口捕获"抓取程序的输出预览窗口 |
| **NDI** | 安装 NDI Runtime 后，OBS 可通过 NDI 源直接接收（零延迟本地传输） |

### 编码规则

每灯占 2 个 32×32 像素色块，纵向排列：

**Block A（角度）：** 整块纯色
- R 通道 = Tilt 编码 = `(tilt + 180) / 360 × 255`
- G 通道 = Pan 编码 = `(pan + 180) / 360 × 255`
- B 通道 = 0（保留）

**Block B（颜色）：** 整块纯色
- RGB 直接映射灯光颜色，已乘以 Dimmer 系数
- 频闪开启时：按频率在颜色和黑色之间切换

### VRChat 端 Udon 采样坐标
