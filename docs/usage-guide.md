## 启动与连接

### 桌面端启动

```bash
pnpm tauri dev      # 开发模式
pnpm tauri build    # 构建发布版
```

启动后程序会：
1. 加载上次保存的灯光状态（如果有）
2. 启动 1280x720 帧合成器（60fps 循环渲染）
3. 在端口 **9000** 启动内嵌 Web 服务器
4. 启动 NDI 输出线程（需要 NDI Runtime）
5. 每 10 秒自动保存当前状态到本地 JSON

### 远端访问

同一局域网内任意设备打开浏览器，访问：

```
http://<运行软件的电脑IP>:9000
```

即可打开与桌面端完全一致的控制界面。多人可同时操作。

## 左侧面板：灯具选择与编组

左侧面板纵向分为两部分，中间有细线分隔。

### Fixtures（灯具选择网格）

网格排列的按钮，每个代表一个灯具。

| 操作 | 说明 |
|------|------|
| **单击灯具按钮** | 选中该灯具（取消其他选中），按钮边框变为红色 |
| **Ctrl+单击** | 多选/取消选中某个灯具（保持已选的不变） |
| **ALL 按钮** | 选中所有灯具 |

每个灯具按钮上显示：
- **编号**：灯具序号（1, 2, 3...）
- **色点**：右侧小圆点显示该灯当前颜色

选中的灯具会成为所有控制操作的目标——改颜色、改角度、改亮度都作用于选中灯具。

### Groups（编组面板）

位于灯具选择下方。

| 元素/操作 | 说明 |
|-----------|------|
| **+ 按钮** | 展开创建编组的对话框 |
| **Group name 输入框** | 输入新编组的名称 |
| **Create 按钮** | 以当前名称和当前选中的灯具创建一个编组 |
| **编组条目** | 点击 = 选中该组内所有灯具 |
| **编组数字** | 显示该组包含的灯具数量 |
| **× 按钮** | 删除该编组 |

编组的用途：快速批量选中一组灯具。比如创建一个"前排灯"组包含灯 1-4，以后点一下就能同时控制这 4 盏灯。

## Tab 页：控制 (Control)

默认激活的页面，包含灯光的核心参数控制。

### XY Pad（Pan/Tilt 触摸板）

左侧的正方形触摸区域。

| 属性 | 说明 |
|------|------|
| **X 轴** | Pan（水平扫描），左端 = -180°，右端 = +180° |
| **Y 轴** | Tilt（垂直扫描），上端 = +180°，下端 = -180° |
| **中心交叉线** | 0° / 0° 参考线 |
| **十字光标** | 当前选中灯具的 Pan/Tilt 位置 |
| **角落度数** | 四角标注最大/最小度数 |

**操作方式：**
- 在 Pad 区域内拖动手指/鼠标 → 实时修改选中灯具的 Pan 和 Tilt
- 支持触摸屏直接拖动，拖动时会阻止页面滚动

### Color Wheel（色轮）

中间的圆形颜色选择器。

| 部分 | 说明 |
|------|------|
| **外圈色环** | 色相（Hue）0°~360°，拖动选择色相 |
| **内部方块** | 饱和度（水平，左低右高）和明度（垂直，上亮下暗） |
| **色相指示器** | 外圈上的白色小圆圈，标记当前色相位置 |
| **SV 指示器** | 内部方块上的白色小圆圈，标记当前饱和度和明度 |
| **右下预览点** | 圆形小色块，显示最终选定的颜色 |

**操作方式：**
- 拖动外圈 → 改变色相
- 拖动内部方块 → 改变饱和度和明度
- 颜色变化实时应用到选中的所有灯具

### Dimmer 推子（亮度）

右侧的竖向滑条。

| 属性 | 说明 |
|------|------|
| **范围** | 0（最暗/关灯）到 1（全亮） |
| **数值显示** | 顶部显示当前数值 |
| **标签** | 底部显示 "Dimmer" |
| **填充色** | 红色填充从底部到当前值 |

**操作方式：**
- 上下拖动手柄（水平白色横条）调整亮度
- Dimmer 乘到 RGB 颜色上输出——Dimmer = 0.5 且颜色纯白时，输出为 (128, 128, 128)

### 旋钮行（Encoder Knobs）

控制区下方的一行旋钮。

| 旋钮 | 功能 | 范围 |
|------|------|------|
| **Pan** | 水平扫描角度精确调节 | -180° ~ +180° |
| **Tilt** | 垂直扫描角度精确调节 | -180° ~ +180° |
| **Dim %** | 亮度百分比 | 0% ~ 100% |
| **Strobe** | 频闪速度（Hz） | 0 ~ 30 |

**操作方式：**
- 在旋钮上**上下拖动**改变数值（向上 = 增大，向下 = 减小）
- 桌面端可用**鼠标滚轮**微调
- 旋钮上的指示线表示当前值在范围内的位置

### Strobe ON/OFF 按钮

位于 Strobe 旋钮下方。

| 状态 | 外观 | 说明 |
|------|------|------|
| **OFF** | 灰色按钮 | 频闪关闭，灯光持续亮 |
| **ON** | 红色按钮 | 频闪开启，灯光按 Strobe 旋钮设定的速度闪烁 |

频闪逻辑在软件端实现：开启后输出画面中该灯的 Block B 会在正常颜色和黑色之间按频率切换。

## Tab 页：效果 (Effects)

用于添加自动化灯光效果。

### 效果网格

5 个效果按钮，点击激活/取消：

| 效果 | 说明 |
|------|------|
| **Color Cycle** | 颜色循环：色相按时间连续旋转，产生彩虹流动效果 |
| **Scan** | 扫描：亮度在全亮与全灭间正弦波动，像探照灯扫描 |
| **Pulse** | 脉冲：亮度按绝对值正弦波动，节奏感强 |
| **Wave** | 波浪：亮度柔和起伏，多灯间有相位差时产生波浪效果 |
| **Random** | 随机：亮度随机跳变，营造闪烁/派对氛围 |

- 按钮高亮 = 当前生效的效果
- 再次点击同一效果 = 关闭效果
- 同时只能有一个效果生效

### 效果参数

| 参数 | 说明 | 范围 |
|------|------|------|
| **Speed** | 效果速度。越大变化越快 | 0.1 ~ 10.0 |
| **Intensity** | 效果强度。0 = 无影响，1 = 完全受效果控制 | 0% ~ 100% |
| **Phase** | 相位偏移。多灯间的相位差，产生追逐/波浪效果 | 0 ~ 6.28 (2π) |

**Phase 说明：** 当 Phase = 0 时，所有灯同步变化。Phase 增大时，灯具间产生递进延迟——灯 1 先变，灯 2 稍后变，以此类推，形成追逐效果。

## Tab 页：Cue 列表

用于保存和调用场景快照。

### 顶部控制条

| 元素 | 说明 |
|------|------|
| **Cue name 输入框** | 输入新 Cue 的名称（可不填，自动命名为 "Cue N"） |
| **Fade 数字输入** | 淡入淡出时间（秒），默认 1.0s |
| **SAVE 按钮** | 将当前所有灯具的完整状态保存为一条 Cue |

### Cue 列表项

每条 Cue 显示：

| 元素 | 说明 |
|------|------|
| **GO 按钮** | 触发该 Cue——将所有灯具状态切换到 Cue 记录的快照 |
| **Cue 名称** | 保存时设定的名称 |
| **Fade 时间** | 显示该 Cue 的淡入淡出时间 |
| **× 按钮** | 删除该 Cue |

当前激活的 Cue 显示红色边框。

**工作流程：**
1. 调好灯光 → 输入名称 → 点 SAVE
2. 调好另一组灯光 → 保存第二个 Cue
3. 演出时点 GO 按钮在不同场景间切换

## Tab 页：输出预览 (Preview)

实时预览输出到 VRChat 的 1280x720 画布。

- 显示帧合成器生成的完整输出图像
- 像素级精确预览（开启 pixelated 渲染模式）
- 可以看到每个灯具的 Block A（角度编码）和 Block B（颜色编码）色块
- 画布宽高比固定 16:9

## 设置页

点击顶部右侧 **设置** 按钮打开设置弹窗。

| 项目 | 说明 |
|------|------|
| 灯具数量 | 1~32，修改后会重新计算输出块布局并尽量填满 720 高度 |
| 输出帧率 | 30 / 60 |
| NDI 启用 | 开关 NDI 输出线程 |
| NDI 名称 | NDI 发送端名称 |
| Spout2 启用 | 开关 Spout2 输出（仅 Windows 有效） |
| Spout2 名称 | Spout2 发送端名称 |
| Web 端口 | 远程 WebUI 端口 |
| 输出窗口可见 | 显示/隐藏 1280x720 独立输出窗口 |

弹窗底部按钮：
- **保存**：写入配置并生效
- **重置全部**：恢复默认状态
- **导出配置**：导出到程序同目录下的 `lighting-state-export.json`
- **导入配置**：从程序同目录下的 `lighting-state-export.json` 导入

## 独立输出窗口

程序会同时打开一个单独的 `1280x720` 输出窗口（标题：`VRC Lighting Output`）：

- 用于 OBS 窗口捕获
- 与主界面共享同一份灯光状态
- 可在设置页或顶部“输出窗”按钮中控制显示/隐藏

---

## 底部推子条

位于窗口最底部，水平排列的推子组。

| 推子 | 说明 |
|------|------|
| **1 ~ 8** | 前 8 个灯具的独立 Dimmer 推子，可直接调节单个灯的亮度 |
| **M（Master）** | 主推子，控制当前选中灯具的亮度。左侧有红色竖线分隔 |

- 推子条可横向滑动（灯具多时）
- 每个推子宽度 56px，触控友好
- 向上拖 = 增大亮度，向下拖 = 减小亮度

## 远程控制 (WebUI)

### 连接方式

1. 确保控制端电脑和远程设备在同一局域网
2. 在远程设备的浏览器中输入 `http://<控制端IP>:9000`
3. 页面加载后自动通过 WebSocket 连接
4. 连接成功后顶部状态栏显示 "Connected"

### 多人协作

- 多个设备可同时连接和操作
- 冲突策略：**后写覆盖**（Last-write-wins），最后一个修改的人生效
- 所有客户端实时同步——一端修改，所有端立即看到变化
- 顶部状态栏显示当前连接的客户端数量

### 与桌面端的区别

WebUI 与桌面端使用**完全相同的 UI 代码**。唯一的区别：
- 桌面端通过 Tauri IPC 直接调用 Rust 后端
- WebUI 通过 WebSocket 发送命令
- 输出预览在 WebUI 端不可用（帧数据通过 IPC 传输，WebSocket 不传输帧）

## 输出协议与 VRChat 接入

### 输出方式

| 方式 | 说明 |
|------|------|
| **窗口捕获** | OBS 使用"窗口捕获"抓取程序的输出预览窗口 |
| **NDI** | 安装 NDI Runtime 后，OBS 可通过 NDI 源直接接收（零延迟本地传输） |

### 编码规则

每灯占 2 个 32×32 像素色块，纵向排列：

**Block A（角度）：** 整块纯色
- R 通道 = Tilt 编码 = `(tilt + 180) / 360 × 255`
- G 通道 = Pan 编码 = `(pan + 180) / 360 × 255`
- B 通道 = 0（保留）

**Block B（颜色）：** 整块纯色
- RGB 直接映射灯光颜色，已乘以 Dimmer 系数
- 频闪开启时：按频率在颜色和黑色之间切换

### VRChat 端 Udon 采样坐标

灯 i（从 0 开始）的采样中心像素：
- Block A 中心：`(18, 18 + i × 34)`
- Block B 中心：`(52, 18 + i × 34)`

UV 坐标（纹理坐标 Y 轴翻转）：
- Block A UV：`(18/1280, 1 - (18 + i×34)/720)`
- Block B UV：`(52/1280, 1 - (18 + i×34)/720)`

详细协议规格见 [protocol.md](protocol.md)。

## 状态持久化

- 所有灯具参数、编组、Cue 列表、效果设置每 **10 秒** 自动保存
- 保存位置：程序同目录下的 `state.json`
- 下次启动时自动恢复
